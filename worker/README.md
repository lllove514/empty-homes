# Ask-the-data backend (Cloudflare Worker)

The public demo runs on GitHub Pages, which serves files but can't run code. This Worker is what lets the demo answer questions anyway: it holds the Anthropic API key (in Worker secrets, never in the browser), runs the same grounded tool loop as `server/ask.py`, and reads the parcel database from Cloudflare D1, which is SQLite hosted inside Workers.

The grounding rules are identical to the Python server, ported line for line:

- The model can only touch the data through three read-only tools: `search_parcels`, `get_parcel`, `get_owner`.
- Every citation in the answer is checked against the ids that actually came out of this request's tool results. One unverifiable citation withholds the whole answer.
- An answer with no verified citation at all is never released either. That closes the free-chat hole: whatever the model was talked into saying, if it isn't anchored to the database, the caller gets a canned refusal instead. `test_grounding.mjs` proves both gates fail closed, offline.

On top of that: single-turn only (no conversation to steer), questions capped at 500 characters, answers at 1000 tokens, 6 questions per minute per IP, and a hard 300-questions-per-day budget so the demo can't be used to drain the API credit.

## Files

- `wrangler.jsonc`: Worker config and the D1 binding. Paste your database id in after creating it.
- `index.js`: the whole Worker. CORS, rate limits, D1 tools, tool loop, citation verifier.
- `test_grounding.mjs`: offline self-check, `node worker/test_grounding.mjs`. No key, no account needed.
- `d1_import.sql`: generated by `python3 pipeline/export_d1.py`, not committed (it's 14 MB of INSERTs).

## Deploy

One-time setup:

```sh
cd worker
npx wrangler login
npx wrangler d1 create empty-homes        # prints a database_id
# paste the database_id into wrangler.jsonc
npx wrangler secret put ANTHROPIC_API_KEY
```

Every time the data pipeline rebuilds:

```sh
python3 ../pipeline/export_d1.py
npx wrangler d1 execute empty-homes --remote --file d1_import.sql
```

Deploy the Worker itself:

```sh
npx wrangler deploy
```

The deployed URL goes in `ENDPOINT` at the top of `web/ask.js`. Locally the site never touches the Worker; the Python server answers at `api/ask` with the same logic.

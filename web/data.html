Data and method

Everything on this site is assembled from four public City of Philadelphia datasets on OpenDataPhilly: the L&I Vacant Property Indicators (the parcel spine), the OPA property assessment file (ownership, values, mailing addresses), the Department of Revenue's parcel-level real estate tax delinquency file, and open L&I violations. Parcels join across all four on the OPA account number.

The method, in five steps. Fetch: each dataset is pulled from the city's API with the schema validated at fetch time, so drift fails loudly instead of silently. Join: everything lands in one SQLite database, one row per likely-vacant parcel. Resolve: raw owner strings are normalized and grouped; public agencies resolve through a hand-curated alias list you can read in the repository (pipeline/agencies.json); non-public owners that share a mailing address are linked as a network. Score: every parcel gets a transparent accountability score, computed as min(years tax-delinquent, 10) + min(open violations, 5) + 3 if publicly owned + 2 if flagged for sheriff sale. Verify: a check script re-derives totals from the raw files and spot-checks parcels against the city's live API before anything is published.

Honest limitations

The city stopped publishing parcel-level tax delinquency in June 2022, so every delinquency fact here is that snapshot and is labeled as such. "Likely vacant" is the city's model-based indicator, not a field inspection; this site inherits its false positives and never says "abandoned". A shared mailing address between two owners is a documented fact, not proof of common control. Owners, including private individuals, appear exactly as the public record names them; nothing is added beyond what the city publishes.

All source data is subject to the City of Philadelphia's open data terms of use. This project's own code and cleaned dataset are free to reuse; the method travels to any city with parcel-level open data.
